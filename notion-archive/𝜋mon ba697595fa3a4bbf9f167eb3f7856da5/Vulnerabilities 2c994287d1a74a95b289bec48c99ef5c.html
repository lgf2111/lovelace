Flutter Security
  • Enable Code Obfuscation
    • Encryption of plaintext data
    • Ensure that functions and class names that are being used are hidden
    • Make use of the -obfuscate parameter
  • Securing of API Keys
    • Apply restriction controls in order to control which applications, websites, or IP addresses are able to access API Keys
    • Encrypting API Keys on runtime
    • Disable tracking of API Keys on the repository
  • Flutter Jailbreak Detection
    • Detect whether the user device is jailbroken
    • Make use of Rootbeer (Android), DTTJailbreakDetection (iOS)
    • A compromised or rooted device might be able to have escalated privileges and would enable the installation of malware and virus
  • Ensuring the use of secure network connections
    • Engage the use of Transport Secure Layer for connection to Flask
  • Ensure that user data is secure
    • Use secure storage for PII, Auth Token, or other sensitive data
    • Employ the use of Flutter_secure_storage
    • Makes use of Keystore for Android and Keychains for iOS
  • Background Snapshot Protection
    • Ensure that background app switchers do not display the last snapshot state of an app
  • Implement Local Authentication
    • Making use of passwords for identification
    • Making use of Biometric systems
    • Engaging the use of 2FA

MongoDB Security
  • Network configuration
    • Use TLS
    • Firewall
    • VPN between DB and API
    • Allow only trusted clients to access network interface and ports
    • Disable direct SSH root access
    • security.clusterIpSourceAllowlist configuration
      setting
    • net.bindIp configuration setting
    • authenticationRestrictions field to the
      db.createUser() command to specify a per-user IP
      allow list
  • Encryption
    • Client-Side Field Level Encryption
    • Encrypt MongoDB data includes log file, data file, config file, keyfiles
  • Logging

    Centralize logging

  • User management
    • Root user > make an account for each application using db
    • Enable access control (Role BAC, instruction on how to is in checklist)
  • Field Level Redaction
  • Authentication using SCRAM (default) or X.509
  • Run MongoDB processes with a dedicated operating system user
    account (not sure how this will work on cloud)
  • Enable input validation (enabled by default)
  • If not using any of this javascript code: mapReduce, $where,
    $accumulator, and $function
    disable server side scripting using
    the --noscripting option
AWS Cloud Security
  • Identity and Access Management
    • AWS IAM Users and Groups
      • A centralised mechanism for creating and managing individual users and their permissions with AWS account
  • Infrastructure Protection
    • Protection via Isolation
      • Amazon VPC (Virtual Private Cloud)
        • Allows one to isolate AWS resources in the cloud
        • Enables one to launch resources into a virtual network
        • Most common VPC features
          • Subnet Routing
          • Network ACLs (Access Control Lists)
          • Security Group
    • Application and OS Security
      • AWS Systems Manager
        • Automate management tasks such as collecting system inventory, applying operating system patches, maintaining up-to-date anti-virus definitions, and configuring operating systems and applications at scale.
        • Features
          • Automation
          • Inventory
          • Patch Manager
          • Parameter Store
          • Run Command
          • Session Manager
  • DDoS
    • Amazon Route 53
      • Hosted at numerous AWS edge locations, creating a global surface area capable of absorbing large amounts of DDoS traffic
    • Amazon CloudFront
      • Only accepts HTTPS and HTTP well-formed connections to prevent many common DDoS attacks
    • AWS Shield
      • A managed DDoS protection service that safeguards web applications that run on AWS.
    • AWS WAF (Web Application Firewall)
      • Helps protect web application from common web exploits
      • Gives control over which traffic to allow or block by defining customisable web security rules
  • Data Protection
    • Protection at Rest
      • Two topes of encryption options available
        • Client-side encryption
        • Server-side encryption
    • Protection in Transit
      • AWS services provide HTTPS endpoints using TLS for communication
      • Use AWS to generate, deploy, and manage public and private certifications used for TLS encryption in web-based workloads
      • Use IPsec with VPN connectivity into AWS to facilitate encryption of traffic
    • Amazon Macie
      • Uses machine learning to automatically discover, classify, and protect sensitive data in AWS
      • Provide dashboards and alerts that give visibility into how datas are being accessed or moved
